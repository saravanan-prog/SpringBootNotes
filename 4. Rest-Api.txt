What is a REST API?
====================

    REST (Representational State Transfer) is a design pattern for building web services 
    that use standard HTTP methods like:

        GET – Retrieve data
        POST – Create new data
        PUT – Update existing data
        DELETE – Delete data

        --------------------------------------------------------------------------    
        | Action        | Method | URL            | Body (JSON)                   |
        | ------------- | ------ | -------------- | ----------------------------- |
        | Get all users | GET    | `/api/users`   | -                             |
        | Get user      | GET    | `/api/users/1` | -                             |
        | Create user   | POST   | `/api/users`   | `{ "name": "Alice" }`         |
        | Update user   | PUT    | `/api/users/1` | `{ "name": "Updated Alice" }` |
        | Delete user   | DELETE | `/api/users/1` | -                             |
        --------------------------------------------------------------------------|

    A REST API in Spring Boot: Uses standard HTTP methods: GET, POST, PUT, DELETE

    1. HTTP Method Annotations

        |--------------------------------------------------------------
        | Annotation       | HTTP Method | Purpose                     |
        | ---------------- | ----------- | --------------------------- |
        | `@GetMapping`    | GET         | Fetch data                  |
        | `@PostMapping`   | POST        | Create new resource         |
        | `@PutMapping`    | PUT         | Update an existing resource |
        | `@DeleteMapping` | DELETE      | Delete a resource           |
        | `@PatchMapping`  | PATCH       | Partially update a resource |
        |---------------------------------------------------------------
       
      
    2. Parameter Annotations

        -------------------------------------------------------------------------------
        | Annotation       | Used For                   | Example Use                 |
        | ---------------- | -------------------------- | --------------------------- |
        | `@PathVariable`  | Extract from URL path      | `/users/{id}`               |
        | `@RequestParam`  | Extract from query string  | `/users?active=true`        |
        | `@RequestBody`   | Deserialize JSON from body | POST/PUT with JSON          |
        | `@RequestHeader` | Access HTTP headers        | Authorization, Content-Type |
        |------------------------------------------------------------------------------ 

    
    3. Summary Table

    |---------------------------------------------------------------
    | Annotation          | Purpose                                |
    | ------------------- | -------------------------------------- |
    | `@RestController`   | Define a REST controller               |
    | `@RequestMapping`   | Base path for controller               |
    | `@GetMapping`       | Handle HTTP GET requests               |
    | `@PostMapping`      | Handle HTTP POST requests              |
    | `@PutMapping`       | Handle HTTP PUT requests               |
    | `@DeleteMapping`    | Handle HTTP DELETE requests            |
    | `@PathVariable`     | Bind URL path variable to method param |
    | `@RequestParam`     | Bind query param to method param       |
    | `@RequestBody`      | Bind JSON request body to object       |
    | `@RequestHeader`    | Access HTTP request headers            |
    | `@ResponseStatus`   | Customize HTTP status code             |
    | `@ExceptionHandler` | Handle exceptions cleanly              |
    | `@Valid`            | Trigger validation on request body     |
    |---------------------------------------------------------------
